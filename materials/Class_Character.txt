クラス名：Character
主担当者名：佐藤　晃汰

属性（変数）一覧：
id:int　キャラクター識別ID
name:String　キャラクター名
positionX:float/potisionY:float　現在の座標　
direction:Enum　現在向いてる方向
speed:float　移動速度
hp:int　体力、通常は1。アイテムや状態によっては複数書き耐えられるように備える
isAlive:
boolean:生存フラグ、falseになったら死亡演出へ
isInvincible:boolean　無敵状態フラグ、ダメージ後の点滅時間用

メソッド（関数）
move(float deltaX, float deltaY):座標更新
takeDamage(int damage): ダメージを受ける処理。hp を減らし、0以下なら die() を呼ぶ。無敵時間中はダメージを無効化する。
die(): 死亡処理。生存フラグを false にし、死亡アニメーションを再生したり、ステージから自身を削除するよう Stage クラスに通知する。
getGridPosition(): 現在の座標から、マップ上の何行・何列目（マス目）にいるかを計算して返す。


子クラス名：Player
主担当者名：佐藤　晃汰

属性（変数）一覧：
playerNumber: int - 1P、2Pなどの識別用
maxBombs: int - 同時に設置できる爆弾の最大数（初期値: 1）
currentBombs: int - 現在フィールドに設置している自分の爆弾の数
bombFirePower: int - 設置する爆弾の火力のマス数（初期値: 2）
score: int - 敵を倒した数やアイテム取得によるスコア

メソッド（関数）：
handleInput(): キーボードやコントローラーの入力を監視し、移動メソッド（move）を呼び出す。
placeBomb(): 自分がいるマスに爆弾を設置する。
※ロジック例: currentBombs < maxBombs かつ 「足元にすでに爆弾がない」場合のみ、Stage に爆弾オブジェクトを追加し、currentBombs を +1 する。
onBombExploded(): 自分の置いた爆弾が爆発したときに、currentBombs を -1 する（再び置けるようにするため）。
pickUpItem(Item item): アイテムに触れたときの処理。
※ロジック例: item.getType() に応じて、maxBombs++ や bombFirePower++、speed += 0.5 などの強化を行う。


子クラス名：Enemy

属性（変数）一覧：
enemyType: Enum - 敵の種類（ザコ敵、高速移動型、壁通過型、ボスなど）
aiPattern: Enum - 行動アルゴリズム（ランダム移動、プレイヤー追跡、爆風回避など）
thinkTimer: float - 次の行動（曲がる、追跡し直すなど）を決めるまでの時間

メソッド（関数）:
updateAI(Stage stage, Player player): 毎フレーム、または一定時間ごとに呼び出され、ステージの状況やプレイヤーの位置を見て次の行動を決める。
※ロジック例: 「前方に壁がある、または現在のマスが爆風の予測範囲内なら、安全な別の方向へ曲がる」という計算を行う。

開発上での注意点

1.壁との衝突（移動制限）:
Character.move() の中で、移動先のマスが stage.isWalkable(x, y) かどうかをチェックし、通れない壁なら座標の更新をストップさせます。
2.爆風との接触（ダメージ）:
爆弾が爆発して発生した「爆風（Blast）オブジェクト」の側から、範囲内にいるキャラクターの takeDamage() を呼び出すようにするとスッキリします（仕様にある「自分もダメージを受ける」は、プレイヤーも敵も同じ Character なので、区別なくダメージ処理に巻き込めます）。
